1.前言

当下微服务技术如火如荼，而不管是实现微服务还是分布式集群都需要依赖容器技术将应用容器化。这是一篇入门教程和使用教程，会尽可能减少概念输出而以通俗的易懂的文字阐述。

1.1 什么是容器技术

  容器技术是虚拟化技术技术的一种，最常见的虚拟化技术莫过于虚拟机（如VMware）。容器的实现要依赖一个叫做容器引擎的东西，这样就只需要将必要的依赖打包到镜像，而不必将整个操作系统打包。如此，容器比虚拟机轻量很多。需要注意的是Docker只是容器化技术的一种实现，只是当下Docker是最火的，因此Docker成了容器化技术的代名词。准确地说，我们常使用的Docker工具是一个容器引擎。

2.Docker概念

  Docker中最重要的概念是：镜像、容器和仓库。这几个概念很重要，但不必急于吃透，用一边就明白了。

2.1 镜像（Image）

  镜像是容纳依赖，代码和配置文件的东西。当我们完成项目开发，就可以通过Docker将项目打包成一个镜像，然后传到服务器上通过Docker直接运行。这里只需要知道Docker镜像是一个将项目封装起来的文件即可。

2.2 容器（Container）

  通过Docker可以将镜像运行起来，运行镜像后会产生容器。类比Windows，镜像相当于安装包，容器相当于安装好的程序，有运行中，退出和出错等状态。

2.3 仓库（Repository）

  存镜像的地方，Docker有官方仓库，我们也可以自己搭建私有仓库。

5.4 删除镜像

如果需要根据镜像名删除镜像则必须跟上tag，通常镜像ID等价于name:tag

5.5 运行镜像

  • docker run：运行镜像
  • -d：以后台方式运行
  • -p 8848:80：将容器的80端口映射到宿主机的8848端口
  • –name my-nginx：容器取名为my-nginx
  • nginx:stable：要运行的镜像为nginx:stable

其他参数

  • -v：用于挂载文件
  • –restart=always：允许容器自动启动
  • -u root：以root身份运行容器
  • 还有一些容器自带的配置参数，具体可看Docker Hub
